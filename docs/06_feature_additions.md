# えんまーる 機能追加仕様書

> 作成：タイミー仕様を参考にしたディスカッション（2026-06-06）より整理

---

## 1. 基本設計方針（追加確認事項）

| 方針 | 内容 |
|------|------|
| マッチング方式 | 即時マッチング・面接なし・先着順 |
| 賃金の流れ | 保育園 → 保育士への直接支払い（えんまーるは立替・預かりを行わない） |

---

## 2. 書類管理機能（新規）

### 2.1 応募前充足チェック

- 応募ボタン押下時に必要書類の充足状態を検証する
- 未充足の場合はマイページ内の書類登録ページへ強制遷移する
- 書類が運営に**認証済み**の保育士のみ応募可能とする

### 2.2 必要書類一覧

| 書類 | 対象 | 形式 |
|------|------|------|
| 履歴書相当情報 | 全員必須 | アプリ上の入力フォーム **または** 画像アップロード（選択型） |
| 保育士証 | 全員必須 | PDF または画像アップロード |
| 健康診断書 | 全員必須 | PDF または画像アップロード |
| 検便結果 | 一部の保育園が必須 | PDF または画像アップロード |

> 履歴書は「アプリ上で監査に必要な最低限の必須項目を入力する方法」と「既存の履歴書画像をアップロードする方法」のどちらかを選択できる。

### 2.3 書類認証フロー

1. 保育士がマイページから書類をアップロード／入力
2. 保育士側には「資料確認中」ステータスが表示される
3. 運営が管理画面でステータスを確認し、認証を実施
4. 認証完了後に保育士の応募が解放される

### 2.4 将来対応予定

- 各書類のサンプル画像・記入例をマイページ内に表示するエリアを設ける

---

## 3. グループ（お気に入り）機能（新規）

- 保育園が過去にマッチングした保育士をグループ登録できる
- 次回募集時にグループ限定で優先公開ができる（全公開との切り替え）
- 継続・長期雇用への発展も想定内の設計とする

> 公開範囲設定（全公開 → グループ限定の切り替えUI）の詳細仕様は別途ディスカッション予定

---

## 4. 打刻機能（新規）

- QRコードなし、双方のアプリ操作で業務開始・終了を確認する
- 業務開始**60分前**にLINE Messaging APIで打刻URLを保育士へ自動送信する
- 打刻URLを開いた後のUI/UXフローの詳細仕様は別途ディスカッション予定

---

## 5. キャンセルポリシー（未設計・残タスク）

保育士側・保育園側それぞれのキャンセルルールおよびペナルティを設計する。

### 設計時の前提条件

- 賃金は保育園→保育士の直接支払いのため、えんまーる側での金銭補償は介在しない
- 保育現場の特性上、急なキャンセルは現場の人員配置に直結するため慎重に設計する
- タイミーのポイント制を参考にしつつ、保育士という資格職の特性とえんまーるの信頼重視の方針を考慮する

### 未決定の論点

- 保育士側・保育園側それぞれのキャンセル可能期限とペナルティの内容
- 当日キャンセル時の応募制限期間
- 無断欠勤の定義と自動検知タイミング（例：打刻URLを業務開始後N分経過しても操作なしの場合）
- ペナルティの実効性をどう担保するか（金銭補償が介在しない前提で）

## 6. LINEログイン・通知機能（新規）

### 6.1 LINEログイン

- LINEアカウントでのソーシャルログインを導入する
- 既存のメールアドレス認証と併用する形で提供する

### 6.2 通知機能

| 通知手段 | 用途 |
|---------|------|
| LINE Messaging API | メインの通知チャネル（打刻URL送付・マッチング通知など） |
| メール | LINEの補完チャネル（LINE未連携ユーザーへの対応・重要な連絡） |

> 要件定義書（01_requirements.md）に既定のLINE連携方針（登録時に公式LINEの友だち追加が必須・月200通の無料枠）を前提とする。
> 通知の具体的なトリガー・文面・送信タイミングの詳細仕様は別途ディスカッション予定。

---

## 7. チャット機能（仕様確定）

- 基本設計書（02_basic_design.md）の `Message` エンティティを実装する
- **利用可能期間：マッチング成立後 〜 業務完了後24時間まで**
- 期間外はチャット画面への遷移・送信ともに不可とする

---

## 8. 募集の掲載期間・定員設定（新規）

- 保育園が募集作成時に掲載期間と定員を設定できる
- 定員に達した時点で自動的に募集を締め切る
- 掲載期限を過ぎた時点でも自動的に募集を締め切る
- 保育園側の管理画面から任意のタイミングで手動締め切りも可能
- 一度締め切った募集の再掲載は不可とする（タイミー準拠）

| 項目 | 内容 |
|------|------|
| 掲載期間 | 開始日時・終了日時を設定（終了日時到来で自動締め切り） |
| 定員 | 募集人数を設定（充足で自動締め切り） |
| 手動締め切り | 管理画面から任意に締め切り可能 |
| 再掲載 | 不可 |
