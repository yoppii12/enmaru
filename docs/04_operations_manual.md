# えんまーる 運用マニュアル

> 事務局（合同会社KASUMIN）向け運用ガイド

---

## 1. 本マニュアルの目的

| 項目 | 内容 |
|------|------|
| 目的 | マッチング〜評価までを迷いなく運用する |
| 対象 | えんまーる事務局（合同会社KASUMIN） |
| 中核 | matchesシート（管理画面のマッチング管理） |

## 2. 事務局の基本スタンス

- **人が介在する安心感を最優先**
- 通知はシステムが自動送信（LINE Messaging API）。事務局の手動送信は不要
- 「急がせない・煽らない・断りやすく」

---

## 3. マッチングステータス定義

| status | 意味 | 事務局の主な役割 |
|--------|------|----------------|
| applied | 応募が入った | 内容確認・連絡 |
| screening | 確認中 | 双方に条件確認 |
| matched | 双方合意 | 日程・条件整理 |
| working | 業務実施中 | 見守り |
| completed | 業務完了 | 評価案内 |
| review_open | 評価受付中 | 回収確認 |
| review_done | 評価完了 | クローズ |

---

## 4. 運用フロー（判断フロー付き）

### STEP 1｜応募発生（status = applied）

**トリガー：** 応募フォームから新規応募

**システム自動通知：**
- 保育園へ：「◯◯さんより応募がありました。内容をご確認いただき、受け入れ可否をサイト上でお知らせください。」
- 管理者へ：新規応募の通知

**事務局対応：**
1. 管理画面のマッチング管理で応募内容を確認
2. 問題があれば公式LINEを通じて調整

---

### STEP 2｜合意成立（status = matched）

**トリガー：** 園・保育士双方がサイト上で合意

**システム自動通知：**
- 双方へ：「マッチングが成立しました🌸 当日の詳細はサイトよりご確認ください。ご不明点があればえんまーる公式LINEへご連絡ください。」

**事務局対応：**
1. 管理画面で matches.status = matched に更新
2. 業務日・時間を確認

---

### STEP 3｜業務実施中（status = working）

**トリガー：** 業務当日を迎えた

**事務局対応：**
1. status = working に更新
2. 当日は基本「見守り」
3. 連絡が来たら即対応

---

### STEP 4｜業務完了（status = completed）

**トリガー：** 業務完了報告フォームの回答あり

**事務局対応：**
1. 両者の完了報告を確認
2. status = completed に更新
3. 評価依頼を送信（review_status = none）

**システム自動通知（双方）：**
- 「本日はありがとうございました。よろしければ、簡単な評価へのご協力をお願いします。（評価ページURL）」

---

### STEP 5｜相互評価回収（review_status管理）

| review_status | 条件 |
|--------------|------|
| none | 両方未提出 |
| partial | どちらか提出 |
| done | 両方提出 |

**事務局対応：**
- 3日経って未提出 → 軽くリマインド
- 内容を必ず確認（次項）

---

### STEP 6｜評価内容チェック（重要判断）

**必ず確認するポイント：**
- 感情的・攻撃的な表現がないか
- 事実と感想が混同されていないか

**NG例：**
- 人格否定
- 園・個人が特定される誹謗

**対応方針：**
- 軽微 → 非公開のまま保管
- 問題あり → admin_memoに記録、必要なら当事者へ連絡

---

### STEP 7｜クローズ（status = review_done）

**条件：** 双方評価提出済・問題なし

**事務局対応：**
1. status = review_done
2. review_status = done
3. admin_memo に総括

---

## 5. トラブル対応

### ケース① ドタキャン
- 公式LINEで双方状況確認（システム通知とは別に事務局から連絡）
- admin_memo に経緯記録
- 評価は無効扱い

### ケース② 評価が辛辣
- 非公開
- 事務局で要約のみ内部利用

### ケース③ 連絡が途絶えた
- 公式LINEから2回まで連絡
- 反応なければクローズ

## 6. 毎日のチェックリスト

| チェック | 見る場所 |
|---------|---------|
| 新規応募 | 管理画面 > マッチング |
| ステータス滞留 | matches（長期間同ステータスのもの） |

| トラブル兆候 | admin_memo |
| 公式LINEへの未返信 | えんまーる公式LINE |

## 7. 評価コメント書き方ガイド

### 保育園 → 保育士 評価

**この評価の目的：** 良し悪しを決めるためではなく、次に活かすため

| 観点 | 良かった点の書き方（例） | 改善につながる書き方（例） |
|------|----------------------|------------------------|
| 勤務態度 | 時間を守り、落ち着いて業務に取り組まれていました | 初日は業務内容を確認しながら進めると安心です |
| 子どもへの関わり | 子ども一人ひとりに丁寧に声をかけていました | 慣れてくると、より積極的に関われそうです |
| コミュニケーション | 職員の声かけに素直に応じていました | 分からないことをもう一歩踏み込んで聞いてもらえると助かります |

**NG表現と言い換え：**

| NG | OK |
|----|-----|
| 向いていないと思いました | 初回としては慣れるまで時間が必要と感じました |
| 動きが悪かった | 役割が分かると、より動きやすくなりそうです |
| 期待外れだった | 今回は経験を積む段階だと感じました |

---

### 保育士 → 保育園 評価

**この評価の目的：** 安心して働ける環境を一緒につくるための声

| 観点 | 良かった点の書き方（例） | 改善につながる書き方（例） |
|------|----------------------|------------------------|
| 受け入れ説明 | 最初に1日の流れを説明いただき安心できました | 事前に簡単な流れを共有いただけると、より落ち着いて動けます |
| 職員の雰囲気 | 職員の方が声をかけてくださり緊張が和らぎました | 初めての立場だとどこまで動いてよいか迷う場面がありました |
| サポート体制 | 困った時にすぐフォローしてもらえました | 忙しい時間帯は声をかけるタイミングに迷いました |

**基本ルール：** 不満ではなく「こうだと助かる」に言い換える

## 8. デモシナリオ（7分）

| フェーズ | 内容 | 時間 |
|---------|------|------|
| 導入 | えんまーるの説明 | 1分 |
| 保育園登録・募集投稿 | 保育園側の操作 | 2分 |
| 保育士登録・応募 | 保育士側の操作 | 2分 |
| マッチング→業務→評価 | 管理画面＋評価フォーム | 2分 |
| まとめ | 締めトーク | 1分 |

**締めトーク：**
> えんまーるは「人手不足を埋める」だけでなく、**良いご縁を育てる仕組み**です。
> 応募→業務→相互評価の一連の流れが、信頼の積み重ねになります。
