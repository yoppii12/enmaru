## 4. 画面遷移（主要導線）

### 4.1 潜在保育士の主要導線

```
トップ(P-01)
  └→ えんまーるとは(P-02)
  └→ 園一覧(P-03) → 園詳細(P-04)
  └→ 新規登録(P-05)
       └→ ログイン(P-06)
            └→ マイページ(U-01)
                 └→ プロフィール編集(U-02)
                 └→ 園検索(U-03) → 園詳細[会員](U-04) → 応募(U-05)
                 └→ 履歴一覧(U-06)
                 └→ 評価入力(U-07) ← 業務完了後のみ
```

### 4.2 保育園の主要導線

```
トップ(P-01)
  └→ 新規登録(P-05) → ログイン(P-06)
       └→ 園マイページ(C-01)
            └→ 園プロフィール編集(C-02)
            └→ 募集情報管理(C-03) → 作成/編集 → 公開
            └→ 応募受信(C-05) → 対応 → 対応履歴(C-06)
            └→ 評価入力(C-07) ← 業務完了後のみ
```

### 4.3 マッチングステータス遷移

```
applied（応募あり）
  └→ screening（確認中）
       └→ matched（合意成立）
            └→ working（業務実施中）
                 └→ completed（業務完了）
                      └→ review_open（評価受付中）
                           └→ review_done（双方評価完了）
```

## 8. 主要画面定義（代表画面）

### P-01 トップページ

**目的：** サービス全体の入口として、理解→行動（登録/ログイン）を促す

**セクション構成：**

| セクション | 内容 |
|----------|------|
| ヘッダー | ロゴ／グローバルナビ（えんまーるとは・保育士・保育園・お問い合わせ） |
| ヒーロー | キャッチコピー「もう一度、保育に関わりたい。その気持ちを大切に」＋二系統CTA |
| 二者向け導線 | 潜在保育士ブロック／保育園ブロック（説明＋CTA） |
| サービス要約 | えんまーるの特徴3点 |
| 安心・信頼 | 相互評価説明／運営体制（KASUMIN） |
| 利用の流れ | 登録→募集→応募→業務→評価の5ステップ |
| CTA集約 | 保育士登録／保育園登録／ログイン |
| フッター | 規約／問い合わせ |

**文言例：**
- ヒーロー：「もう一度、保育に関わりたい。その気持ちを、そっと大切にできる場所。」
- 保育士向け：「あなたのペースで、保育の現場へ」
- 保育園向け：「必要なときに、信頼できる人材と」
- 安心：「相互評価と運営の見守りで、安心して利用できます」

## 9. 通知設計

| タイミング | 宛先 | 手段 | 内容 |
|----------|------|------|------|
| 登録完了 | 利用者 | メール | ウェルカムメール |
| 応募発生 | 保育園 | メール/LINE | 応募通知 |
| マッチング成立 | 双方 | メール/LINE | 成立通知 |
| 業務完了報告受領 | 事務局 | メール | 確認依頼 |
| 評価依頼 | 双方 | メール/LINE | 評価フォームURL |
| 評価完了 | 事務局 | システム | review_status更新 |
