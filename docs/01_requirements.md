## 2. サービス概要

### 2.1 基本情報

| 項目 | 内容 |
|------|------|
| サービス名称 | えんまーる |
| サービス形態 | Webアプリケーション |
| 対象分野 | 保育人材マッチング |
| 主な特徴 | 段階的マッチング＋業務後の相互評価 |

### 2.2 サービスの目的

| 観点 | 内容 |
|------|------|
| 社会課題 | 潜在保育士の増加／保育園の人材不足 |
| 解決方針 | 双方向の情報可視化と関係構築支援 |
| 目指す状態 | 信頼を前提とした持続可能なマッチング |
| 差別化要素 | 実体験に基づく相互評価機能 |

---

## 3. 背景・課題整理

### 3.1 社会的背景

| 項目 | 内容 |
|------|------|
| 人材構造 | 資格保有者と就業者数の乖離 |
| 現場課題 | 慢性的な人手不足 |
| 雇用形態 | フルタイム前提によるミスマッチ |
| 復職障壁 | 心理的・情報的ハードル |

### 3.2 現状の課題

| 課題 | 内容 |
|------|------|
| 情報不足 | 潜在保育士が園を具体的に想像できない |
| 発信不足 | 園の考え方・雰囲気が伝わりにくい |
| 信頼形成 | 実体験に基づく判断材料が少ない |
| 継続性 | ミスマッチによる早期離職 |

---

## 4. 想定ユーザー

### 4.1 ユーザー区分

| 区分 | 主な対象 | 役割 |
|------|---------|------|
| 潜在保育士 | 保育士資格保有者 | 応募・業務実施・評価 |
| 保育園 | 園長・採用担当 | 募集・受け入れ・評価 |
| 管理者 | 合同会社KASUMIN | 運営・監視・改善 |

### 4.2 ユーザー特性

| 区分 | 特性・ニーズ |
|------|------------|
| 潜在保育士 | ブランクあり／柔軟な働き方希望 |
| 保育園 | 人材不足／ミスマッチ回避 |
| 管理者 | 健全運営／信頼性担保 |

---

## 5. 利用シーン（ユースケース）

### 5.1 マッチング〜評価までの流れ

| フェーズ | 潜在保育士 | 保育園 |
|---------|-----------|--------|
| 募集前 | 園情報を閲覧 | 募集情報を掲載 |
| 応募 | 募集に応募 | 応募内容を確認 |
| マッチング | 合意 | 合意 |
| 業務実施 | 業務を行う | 業務を受け入れる |
| 業務完了 | 完了報告 | 完了確認 |
| 評価 | 園を評価 | 保育士を評価 |
| 次回判断 | 評価を参考に応募 | 評価を参考に検討 |

---

## 6. 機能要件

### 6.1 潜在保育士向け機能

| 機能名 | 内容 |
|--------|------|
| 会員登録／ログイン | アカウント作成・認証 |
| プロフィール管理 | 経歴・希望条件登録 |
| 募集情報閲覧 | 保育園の募集内容確認 |
| 応募機能 | 募集情報への応募 |
| 評価機能 | 業務後の保育園評価 |

### 6.2 保育園向け機能

| 機能名 | 内容 |
|--------|------|
| 園アカウント登録 | 園情報登録 |
| 募集情報管理 | 募集条件・内容の掲載 |
| 応募管理 | 応募内容確認・合否管理 |
| 評価機能 | 業務後の保育士評価 |

### 6.3 マッチング・業務管理（共通）

| 機能名 | 内容 |
|--------|------|
| マッチング管理 | 応募〜合意の管理 |
| 業務ステータス | 未実施／実施中／完了 |
| 評価可能判定 | 業務完了後のみ評価可能 |

### 6.4 相互評価機能（コア機能）

#### 6.4.1 保育園 → 潜在保育士 評価

| 項目 | 内容 |
|------|------|
| 評価対象 | 業務を実施した潜在保育士 |
| 評価タイミング | 業務完了後 |
| 評価項目 | 勤務態度・コミュニケーション・業務への姿勢（各1〜5） |
| 評価方法 | 数値評価＋コメント（任意） |
| またお願いしたいか | Yes/No |

#### 6.4.2 潜在保育士 → 保育園 評価

| 項目 | 内容 |
|------|------|
| 評価対象 | 業務を実施した保育園 |
| 評価タイミング | 業務完了後 |
| 評価項目 | 受け入れ説明・現場の雰囲気・サポート体制（各1〜5） |
| 評価方法 | 数値評価＋コメント（任意） |
| また働きたいか | Yes/No |

### 6.5 評価機能の前提条件

| 項目 | 方針 |
|------|------|
| 評価条件 | マッチング＋業務完了後のみ |
| 公平性 | 双方向評価 |
| 匿名性 | 個人特定を防ぐ配慮 |
| 不正防止 | 管理者による監視 |
| 公開範囲 | 初期は非公開（事務局管理）、段階的に公開拡大 |
